Contexto antes de código02
Definición simple

Streamlit convierte Python en una interfaz web

No empezamos con HTML, CSS o JavaScript. Empezamos con el script que ya entiende el equipo.

La promesa

  • Escribes Python.
  • Agregas widgets.
  • Streamlit renderiza la app.
  • Iteras en segundos.
Script primero, app después03
Modelo mental

Cada interacción vuelve a correr el script

1

Usuario cambia widget

Slider, selectbox, checkbox o botón.

2

Streamlit rerun

El archivo se ejecuta de arriba hacia abajo.

3

Python calcula

Pandas filtra, agrupa y resume.

4

La pantalla cambia

Tablas, métricas y gráficas se actualizan.

Si entiendes reruns, entiendes Streamlit04
Primer comando

La app empieza con un archivo Python normal

import streamlit as st

st.title("Mi primera app")
st.write("Hola, datos")

streamlit run streamlit_app.py

Varios widgets en una misma app
El widget regresa un valor de Python06
Patrón central

Widget → variable → filtro → resultado

min_grade = st.slider(
    "Calificación mínima",
    0, 100, 80
)

filtered = data[
    data["calificacion"] >= min_grade
]

st.dataframe(filtered)

No es magia de frontend. Es Pandas reaccionando a valores que una persona controla.

El slider filtra la tabla en tiempo real

Carga y cache

No recargues datos pesados en cada rerun

@st.cache_data
def get_students():
    return load_students(DATA_PATH)

data = get_students()

Cachear no es optimización prematura cuando el archivo crece: es evitar que la app se sienta lenta desde el primer filtro.
